# Acompanhamento de Andamento — Projeto de Agendamento

Este arquivo monitora o progresso das sprints e das tasks. Atualize as marcações `status:` conforme as tarefas avançam.

## Sprints

- Sprint 1 — Inicial: status: `in_progress` # valores: pending | in_progress | completed

## Tasks Sprint 1

- H01 — Formulário público: status: `pending`
- H02 — API criar solicitação: status: `pending`
- H09 — Testes unitários: status: `pending`
- H10 — Observability / envs: status: `pending`

---

Como usar:

- Quando uma task iniciar, atualize `status` para `in_progress`.
- Ao concluir e passar QA, atualize para `completed`.
- Quando todas as tasks da sprint estiverem `completed`, marque a sprint como `completed`.
