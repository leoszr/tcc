# H09.1 — Testes unitários: validadores (dia da semana, CPF/CNPJ)

Objetivo:
Escrever o conjunto de casos de teste que validam as regras de negócio críticas: aceitação de apenas sábados/domingos e validação básica de CPF/CNPJ.

Entradas / Saídas:

- Entrada: funções/serviços de validação (documentados nas tasks H02.3 e H02.4)
- Saída: especificação de testes unitários com cenários e dados de entrada esperados.

Cenários mínimos:

- `validarDia`: sábado → ok
- `validarDia`: domingo → ok
- `validarDia`: segunda-feira → erro
- `validarCPF_CNPJ`: CPF válido → ok
- `validarCPF_CNPJ`: CPF inválido → erro

Critérios de Aceite:

- Arquivo de testes com os cenários acima; testes devem rodar no pipeline local e passar.

Dependências:

- H02.3 (validar dia) e utilitário de validação de CPF/CNPJ (documentar onde será colocado)

Estimativa: 0.5d (4 horas)

Checklist de QA:

- Executar suíte de testes unitários e verificar todos os cenários.
