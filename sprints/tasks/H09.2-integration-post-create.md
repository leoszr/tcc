# H09.2 — Teste de integração mínimo: POST cria reserva e persiste

Objetivo:
Definir o teste de integração que realiza uma chamada HTTP ao endpoint `POST /api/solicitacoes` com payload válido e verifica que o registro foi persistido no banco.

Entradas / Saídas:

- Entrada: payload JSON válido com `data_evento` em sábado/domingo e campos obrigatórios.
- Saída: verificação no banco de que existe um registro com os mesmos campos e `status` = `PENDING`.

Critérios de Aceite:

- Teste de integração automatizado executável em ambiente CI local (in-memory DB ou Postgres de teste) e que passe com resultado OK.

Dependências:

- H01.3 (migration), H02.5 (controller) implementados.

Estimativa: 0.5d (4 horas)

Checklist de QA:

- Executar teste de integração e checar assertivas sobre o banco.
